# CLOAI_DESKTOP_v0.0.3_RELEASE_NOTE

发布日期：2026-04-29
版本：v0.0.3

Cloai Desktop v0.0.3 是一次以“主工作区收拢”和“桌面原生化补强”为重点的预览版更新。这个版本没有追求堆很多新页面，而是优先整理桌面端已经存在的核心路径：把旧的 Cowork 分支收回主导航，把 Code 页面提升为更完整的工作入口，并继续补齐 Tauri 本地桥接能力。

## 重点更新

- 主导航结构收拢：移除 Cowork 独立入口，统一聚焦 Chat / Code 两个主模式，Projects 作为项目上下文入口随模式工作。
- 新增窗口控制组件：桌面端补上更完整的原生窗口控制体验。
- 代码主工作区升级：整合 Code 页面、对话工作区、项目工作区与拖拽分栏交互。
- 原生命令桥接补强：继续完善 commands / streaming / artifacts / projects 等底层能力。
- 版本号同步：`cloai-code` 与 `cloai-desktop` 均升级到 `v0.0.3`。

## 详细说明

### 1. 导航与页面结构调整

这一版的桌面端不再保留 Cowork 作为单独工作区入口。相关页面、图标与侧边栏资源已经移除，顶部模式重新围绕 Chat 和 Code 组织，Projects 则作为项目上下文入口跟随当前模式工作。这样做的目的，是减少预览阶段过早分叉的交互路径，让核心功能先收敛到真正高频的开发流程上。

### 2. 桌面窗口体验增强

新增 `WindowControls` 组件，并同步调整标题栏与 Tauri 窗口配置。桌面端现在在整体操作感上更接近原生应用，而不只是把 Web UI 套进一个壳里。这也为后续继续完善平台相关交互留出了更稳的基础。

### 3. Code 页面成为更完整的主工作区

`CodePage` 在本次更新中承担了更多主界面职责。与 `ConversationWorkspace`、`ProjectsWorkspace`、拖拽分栏组件之间的协同关系也被重新整理，使代码对话、项目上下文和布局调整更自然地发生在同一工作流中。

### 4. Native bridge 持续补强

Rust 侧的 `commands.rs`、`lib.rs`、`streaming.rs`、`projects.rs`、`artifacts.rs` 与 capability 配置均有同步更新。本次重点不是单一暴露某个按钮功能，而是继续把桌面端与底层 CLI / bridge 之间的调用面补齐，为后续更多本地动作、项目能力和产物处理能力做准备。

## 适合关注的人群

如果你已经在使用 cloai-code，但更希望通过桌面端完成模型切换、对话、项目上下文管理和代码工作区协同，那么 v0.0.3 会比前一版更接近“可以连续使用”的状态。特别是对不喜欢在多个页面和入口之间跳转的用户，这一版会明显更顺手。

## 已知说明

- 当前仍属于预览阶段版本。
- Windows `.exe` 仍是当前最直接的分发形态。
- `.dmg` 与 `.AppImage` 已进入打包目标配置，但实际分发节奏仍以 Releases 页面为准。

## 下一步方向

后续版本会继续围绕三个方向推进：
1. 继续完善桌面端本地桥接能力与可调用动作。
2. 继续增强 Code / Projects 主工作区的协同体验。
3. 在保持官方风格一致性的前提下，补齐更适合桌面应用的原生细节。
